홈 화면의 타임라인 문법을 그대로 가져와, 업데이트 페이지에서도 발표 흐름과 구현 방향이 같은 언어로 읽히게 맞췄습니다.

2주차
설명형 UI 방향 설정

데모 증거 정리와 샘플 분석 결과를 묶어, 발표에서 어떤 근거를 먼저 보여줄지 초기 틀을 세웠습니다.

3주차
데이터 수집과 해석 방식 정리

공개 코퍼스 활용, 메일함 분석 결과 해석, 시연 전략을 함께 정리하면서 튜닝 기준을 분리했습니다.

4주차
MVP 범위와 발표 자료 패키지 정리

현재 구현과 향후 과제를 구분해 문서화하고, 발표 흐름과 공개 사이트 구조를 맞춰가기 시작했습니다.

현재
공개 랜딩과 서브페이지 정돈

브랜드 허브형 랜딩, 데모 페이지, 서브페이지 톤 통일을 함께 다듬어 포트폴리오형 사이트로 묶고 있습니다.

방향 전환

최근까지의 방향 전환

01

점수 중심에서 설명 중심으로

단순 점수 출력보다 사용자가 먼저 읽어야 할 근거와 대응 문장을 결과 앞에 배치하는 방향으로 무게중심이 이동했습니다.

02

본문과 HTML 신호 강화

헤더만 보지 않고, 숨은 링크와 리다이렉트, 사회공학 표현을 함께 읽도록 본문/HTML 분석을 보강했습니다.

03

데모와 브랜드 허브 분리

시연용 콘솔과 프로젝트 랜딩을 같은 사이트 안에서 분리해, 설명과 체험이 섞이지 않도록 구조를 정리했습니다.

  • 문서, 발표 자산, 사이트가 같은 프로젝트 언어로 정리된 점
