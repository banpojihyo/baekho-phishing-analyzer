먼저 가려도 되는 정보

  • 수신자 이름과 개인 식별이 쉬운 주소록 이름
  • 상세 주소, 개인 전화번호, 내부 서명
  • 주민번호, 계좌번호, 인증코드, 비밀번호
  • 로그인 가능한 토큰·query가 붙은 긴 URL
분석과 무관한 개인 식별 정보는 먼저 제거해도 됩니다.

남겨두면 분석에 도움이 되는 정보

  • 발신자 주소와 표시 이름
  • 메일 제목, 본문, 첨부 유무 또는 메시지 본문 캡처
  • 의심 링크와 연결 방식
  • 분석에 필요한 헤더, 원문 구조, 대화 맥락 요약
`왜 수상했는지` 한 줄만 덧붙여도 수동 검토 속도가 훨씬 빨라집니다.

업로드 직전 20초 체크리스트

  • 자료 유형이 `.eml`, 링크, 캡처 중 무엇인지 적었는가
  • 수신자 이름, 연락처, 서명은 가렸는가
  • URL에 토큰이나 개인 식별 query가 있으면 제거했는가
  • 받은 채널과 의심 이유를 한 줄로 적었는가
판단에 필요한 구조는 남기고, 식별 정보만 줄이는 방향을 권장합니다.

메일 원문 제출 가이드

이메일 제보는 이렇게 준비하면 됩니다

제보 범위는 넓게 열어 두었지만, 현재 자동 분석과 가장 직접 연결되는 형식은 여전히 .eml입니다. 서비스마다 메뉴 이름은 조금씩 다를 수 있지만, 아래 흐름만 기억하면 `원문 저장` 또는 `메일함 백업` 경로를 찾는 데 도움이 됩니다.

네이버 메일

환경설정 > 메일함관리 > 백업
  1. 환경설정으로 이동

    네이버 메일에 들어간 뒤 `환경설정 > 메일함관리`로 이동합니다.

  2. 원하는 메일함에서 백업

    `받은메일함`, `보낸메일함` 같은 행에서 `백업`을 누르고 zip 파일을 생성합니다.

  3. zip 다운로드 후 확인

    팝업에서 zip을 내려받고 압축을 푼 뒤, 필요한 `.eml`만 골라 보내면 됩니다.

Windows에서는 압축을 푼 폴더에서 `보기 > 파일 확장명`을 켜고, 파일명 끝이 .eml인지 확인하면 됩니다. 보이지 않으면 폴더 검색창에 *.eml을 입력해 확인할 수 있습니다.

Gmail

더보기 > 메시지 다운로드
  1. 메일 1건이면

    Gmail에서 메일을 연 뒤 우측 상단 `더보기` 메뉴를 엽니다.

  2. 바로 저장

    메뉴에서 `메시지 다운로드`를 누르면 현재 메일을 `.eml`로 저장할 수 있습니다.

  3. 메일함 단위면

    `Google Takeout`으로 전체를 내보낸 뒤, 발표용으로 필요한 샘플만 추립니다.

Gmail은 메일 1건을 `.eml`로 바로 내려받을 수 있고, 메일함 전체 내보내기는 `Google Takeout`을 사용하면 됩니다. 메뉴 이름은 계정 언어 설정에 따라 영어 또는 번역된 형태로 보일 수 있습니다.

Outlook · Exchange

다른 이름으로 저장
  1. 메일 열기

    `다른 이름으로 저장`, `메시지 원본 보기`, `다운로드` 계열 메뉴를 찾습니다.

  2. 형식 확인

    가능하면 헤더가 보존되는 형식을 우선 선택합니다. 본문 복사만 보내는 방식은 지양합니다.

  3. 최소 메모 추가

    `언제`, `어떤 채널`, `왜 수상했는지`를 짧게 적으면 수동 검토가 쉬워집니다.

네이버 메일 요약

  1. 메일 화면 열기

    네이버 메인 또는 메일 서비스에서 필요한 메일함으로 이동합니다.

  2. 메일함관리 이동

    환경설정의 `메일함관리`에서 원하는 메일함의 `백업`을 선택합니다.

  3. .eml 확인

    zip을 내려받아 압축을 풀고, 필요한 사례의 .eml만 골라 보냅니다.

Gmail 요약

  1. 메일 1건 선택

    받은편지함에서 제보하려는 메일을 열고 우측 상단 더보기 메뉴를 찾습니다.

  2. 메시지 다운로드

    `메시지 다운로드`를 선택해 현재 메일을 .eml로 저장합니다.

  3. 전체 내보내기

    메일함 단위 내보내기가 필요하면 Google Takeout에서 필요한 범위만 선택합니다.

공개 자료 원칙

실제 계정 화면 캡처는 공개 저장소와 공개 페이지에 올리지 않습니다. 안내 이미지는 추후 더미 계정 또는 비식별 mock 화면으로만 추가합니다.

지금은 텍스트 절차만 제공하고, 민감한 메일함 화면이나 계정 정보가 포함될 수 있는 원본 스크린샷은 제외합니다.
메뉴명이 정확히 일치하지 않을 수 있습니다. 가장 중요한 기준은 `본문만 복사`가 아니라 `헤더와 원문 구조가 남는 저장 방식`을 찾는 것입니다. 메일함 전체를 보내기 부담스럽다면, `대표 샘플 1건만` 보내도 충분합니다. 네이버 메일 백업은 `zip 다운로드 -> 압축 해제 -> .eml 확인` 순서로 이해하면 가장 쉽습니다.

익명 보호 원칙

  • 원문 전체를 공개 자료에 그대로 싣지 않습니다.
  • 개인정보와 로그인 가능한 링크는 비식별 또는 제거를 우선합니다.
  • 요청이 있으면 보관 중인 자료의 삭제를 검토합니다.
  • 이 채널은 수업·연구 목적의 자발적 제보 창구이며, 상용 고객센터나 수사기관 신고 창구는 아닙니다.

제보가 실제로 도움이 되는 방식

  • 광고성 메일이 과탐되는 이유를 줄이는 규칙 튜닝
  • 정상 안내와 회색지대 메일을 구분하는 라벨링 기준 보정
  • 의심 링크와 생활형 사칭 패턴을 발표와 보고서의 실제 사례로 연결
  • 향후 공개 코퍼스로 보이지 않는 한국형 문자·메신저 문맥 보강

현재 운영 기준 보관 정책

  • 원문 파일은 공개 Git 저장소나 공개 발표 자료에 그대로 올리지 않습니다.
  • 분석과 규칙 튜닝에 필요한 동안에만 로컬 비공개 작업 공간에서 검토합니다.
  • 발표와 문서에는 원칙적으로 익명화된 요약, 근거, 집계 결과만 사용합니다.
  • 문자·메신저 제보는 현재 자동 판정이 아니라 수동 검토/향후 보강 자료로 구분합니다.
  • 삭제 요청이 들어오면 해당 자료를 우선 확인하고 제거를 검토합니다.
백호는 상용 고객 데이터 플랫폼이 아니라 연구·발표 검증용 보조 수집 구조를 전제로 운영하고 있습니다. 그래서 `많이 받는 것`보다 `현재 되는 분석과 아직 수동 검토인 범위를 정직하게 설명하는 것`을 우선합니다.

익명 사례 예시

광고성인데 high로 분류된 사례

유통사 프로모션 메일이 `HTTP`, `IP click tracker`, `긴 URL` 때문에 high로 잡힌 적이 있었습니다. 이런 사례는 제보가 들어와야 false positive 패턴을 줄이는 데 직접 도움이 됩니다.
