# 프로젝트 헌장 - PhishShield-Analyzer

## 1. 한 줄 정의

사용자 행동 이전 단계에서 사회공학 기반 위험을 탐지하고,
근거를 설명하는 피싱 분석 시스템을 구축한다.
공개 사이트와 제보 채널은 피싱 전반으로 설계하되,
이번 학기 자동 분석 MVP는 이메일 기반 분석기를 중심으로 완성한다.

## 2. 프로젝트 필요성

- 피싱과 사회공학은 주요 초기 침투 벡터이다.
- 이유가 불명확한 경고는 사용자가 무시하기 쉽다.
- 실무형 1차 선별 도구에 대한 수요가 높다.

## 3. 문제 정의

1. 사용자는 메일/URL/첨부파일의 신뢰도를 빠르게 판단하기 어렵다.
2. 기존 경고는 맥락 설명이 부족해 실제 대응으로 이어지지 않는다.
3. 교육 및 실습용으로 재현 가능한 방어 파이프라인이 필요하다.

## 4. 학기 목표

1. 위험 점수 산출 프로토타입 구현(룰 기반 우선, ML/LLM 선택)
2. 보고서/UI에서 설명 가능한 탐지 근거 제공
3. 주차별 반복 개선으로 성능 향상 근거 확보

## 5. 상위 방향성과 이번 학기 MVP

- 상위 방향성: 이메일, URL, 문자/메신저 등 사회공학형 피싱 입력을 설명 가능한 근거로 분석하는 `설명형 피싱 분석기`
- 공개 사이트/제보 채널: 실제 제보 유입과 사용자 접점을 넓히기 위해 이메일, 의심 링크, 문자/메신저 사례까지 폭넓게 안내
- 이번 학기 자동 분석 MVP: `.eml` 기반 이메일 분석과 URL/첨부파일 설명형 분석 흐름을 가장 먼저 단단하게 구현

## 6. 범위

### 포함 범위

- `.eml` 파싱 및 의심 신호 추출
- URL/도메인 특징 분석
- 첨부파일 정적 선별(유형/해시/의심 확장자)
- 위험 점수 + 근거 보고서 출력
- 공개 사이트에서 피싱 제보 유형별 안내와 비식별 제출 가이드 제공

### 제외 범위 (1학기 기준)

- 문자 스크린샷 OCR 기반 스미싱 자동 분석
- 메신저 대화형 피싱 자동 판정 파이프라인
- 대규모 악성코드 동적 분석 인프라 운영
- 기업 메일 시스템 실서비스 연동
- 완전 자동 차단 정책 배포

## 7. 성공 지표

1. 샘플 데이터셋 기준 탐지 성능 확보(예: F1 >= 0.75)
2. 보고서 생성 지연 시간 목표 달성(예: 10초 이내)
3. 발표용 안정적 이메일 분석 데모 시나리오 3종 확보

## 8. 예상 산출물

1. 소스코드 및 실행 가이드
2. 주차별 진행 기록
3. 최종 기술 보고서 또는 연구 보고서
4. 최종 발표 자료 및 데모 영상

## 9. 리스크 및 대응

1. 데이터 부족
   - 공개 데이터셋 + 자체 샘플 조합 사용
2. 모델 성능 부족
   - 룰 엔진을 기본 기준선으로 유지
3. 일정 지연
   - Must/Should/Could 기반 범위 통제
4. 범위 과확장
   - `설명형 피싱 분석기` 비전과 넓은 제보 채널은 유지하되, 이번 학기 자동 분석 구현과 평가는 이메일 중심 MVP에 한정
