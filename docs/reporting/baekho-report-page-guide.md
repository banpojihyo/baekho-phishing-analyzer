# baekho.app 제보 페이지 운영 가이드

- 작성 기준: 2026-04-03
- 목적: `baekho.app/report`를 신뢰감 있게 운영하고, 제보자가 `보내도 안전한지`를 빠르게 이해할 수 있도록 구조와 문구 기준을 정리

## 1. 왜 제보 페이지에 신뢰 정보가 필요한가

사람들이 제보를 망설이는 이유는 보통 세 가지다.

- `보내도 안전한가`
- `내 정보가 노출되지 않는가`
- `보내면 실제로 의미 있게 쓰이는가`

따라서 제보 페이지는 단순히 `이메일 주소 하나`만 보여주는 구조보다, `처리 절차`, `익명 보호`, `활용 방식`, `사례 예시`를 함께 보여주는 구조가 더 좋다.

## 2. 페이지에 꼭 있어야 하는 섹션

### 2-1. 제보 안내 히어로

- 무엇을 받는지
- 어떤 형식이 좋은지
- 어디로 보내는지
- 동의 기반 수집임을 분명히 쓰기

예시 문장:

`백호는 동의를 전제로 피싱 의심 메일 사례를 제보받고 있습니다. 제출 자료는 연구와 캡스톤 검증 목적으로만 활용하며, 외부 공개가 필요한 경우에는 비식별 처리를 우선합니다.`

### 2-2. 제보 후 처리 절차

권장 4단계:

1. `접수 확인`
2. `비식별 검토`
3. `규칙 검증`
4. `익명 요약 활용`

핵심은 `보내면 어디로 사라지는지 모르겠다`는 불안을 줄여주는 것이다.

### 2-3. 익명 보호 원칙

아래 항목은 구체적으로 적는 편이 좋다.

- 원문 전체 공개 여부
- 개인정보 비식별 여부
- 삭제 요청 가능 여부
- 로그인 가능한 링크, 인증코드 등 민감 항목 처리 원칙

예시 문장:

`원문 전체를 발표 자료나 공개 문서에 그대로 사용하지 않습니다. 이름, 주소, 계정 식별자, 로그인 가능한 링크는 비식별 또는 제거를 우선합니다.`

### 2-4. 실제 도움 방식

후기처럼 보이게 꾸미기보다, `제보가 어디에 쓰이는지`를 정직하게 설명하는 편이 좋다.

예시:

- `광고성 메일 false positive 튜닝`
- `정상 안내 / 회색지대 구분 기준 보정`
- `발표와 보고서에서 실제 사례 기반 근거 제시`

### 2-5. 익명화된 사례 예시

후기가 아직 충분하지 않다면 `익명화된 실제 사례` 또는 `샘플 예시`를 쓰는 편이 낫다.

좋은 예:

- `유통사 프로모션 메일이 high로 분류된 false positive 사례`
- `금융 안내 메일이 회색지대로 분류된 사례`

피해야 하는 예:

- 가짜 후기
- 존재하지 않는 이용자 평가
- `완벽하게 잡아줬다` 같은 과장 문구

## 3. 가짜 후기 대신 무엇을 보여줄 것인가

`구라 후기`는 단기적으로 그럴듯해 보여도, 프로젝트 전체 신뢰를 크게 떨어뜨릴 수 있다.

대신 아래 조합이 더 안전하고 설득력 있다.

- `처리 절차`
- `익명 보호 원칙`
- `익명화된 실제 사례`
- `현재 공개 범위`
- `삭제 요청 / 문의 방법`

즉 신뢰를 `증언`이 아니라 `투명한 운영 방식`으로 설계하는 것이 좋다.

## 4. 문구 톤 가이드

좋은 톤:

- 차분함
- 구체적 설명
- 과장 없는 표현
- 한계도 같이 밝히는 표현

예시:

- `연구와 발표 검증을 위한 보조 수집 구조입니다.`
- `상용 고객지원 채널이나 실시간 침해대응 창구는 아닙니다.`
- `답변과 분석 반영에는 시간이 걸릴 수 있습니다.`

피해야 하는 톤:

- `완벽하게 보호됩니다`
- `모든 피싱을 잡을 수 있습니다`
- `즉시 대응합니다`

## 5. 현재 추천 구조

`baekho.app/report`는 아래 순서로 유지하는 것이 좋다.

1. 히어로
2. 어떤 자료가 도움이 되는지
3. 제출 방법 / 활용 원칙
4. 제보 후 처리 절차
5. 익명 보호 / 실제 도움 방식
6. 익명화된 사례 예시
7. 이메일 템플릿
8. 현재 공개 범위

## 6. 다음에 추가하면 좋은 것

- `자주 묻는 질문`
- `자료 삭제 요청 안내`
- `최근 반영된 익명 사례 수`
- `실제 제보 기반으로 튜닝된 규칙 예시`

## 6-1. 2026-04-10 반영 상태

- `baekho.app/report`에는 서비스별 `메일 내보내기 가이드`가 들어간다.
  - 네이버 메일
  - Gmail
  - Outlook / Exchange
- 공개 저장소와 공개 페이지에는 실제 계정 화면 캡처를 올리지 않는다.
- 안내가 필요하면 텍스트 단계 안내를 우선 사용하고, 이미지는 더미 계정 또는 비식별 mock 화면만 사용한다.
- `개인정보 마스킹 가이드`를 별도 섹션으로 분리해, 가려도 되는 정보 / 남겨야 하는 정보 / 업로드 직전 체크리스트를 함께 보여준다.
- 마스킹 전후 예시를 같이 두어 제보자가 `어디까지 지워도 되는지`를 빠르게 이해할 수 있게 한다.
- 네이버 메일 안내는 실제 확인한 흐름을 텍스트로 정리한다.
  - `환경설정 > 메일함관리`
  - 원하는 메일함의 `백업`
  - zip 다운로드
  - 압축 해제 후 `.eml` 파일 확인
- Gmail 안내는 현재 도움말 기준 경로로 정리한다.
  - 원하는 메일 1건 열기
  - 우측 상단 `더보기`
  - `메시지 다운로드`
  - 메일함 전체 내보내기가 필요하면 `Google Takeout`

## 7. 한 줄 결론

`제보 페이지는 감성 후기보다, 투명한 처리 절차와 익명 보호 원칙, 익명화된 사례 예시를 보여주는 구조가 더 신뢰를 만든다.`

- `/report` 페이지의 메일 내보내기 안내는 공개 안전성을 우선해 텍스트 단계로 유지한다.
- 네이버 메일은 `메일함관리 -> 백업 -> zip 다운로드 -> .eml 확인` 흐름으로 설명한다.
- Gmail은 `메일 열기 -> 더보기 -> 메시지 다운로드` 흐름과 `Google Takeout` 기반 전체 내보내기를 분리해 설명한다.
